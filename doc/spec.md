# 曦仔汉字书写助手规格说明

## 1. 目标

本应用用于帮助儿童学习中文汉字的读音、字形和笔顺。核心使用场景是：输入一句中文，系统拆解出其中的汉字，并支持逐字查看、朗读和书写演示。

## 2. 目标用户

- 儿童学习者
- 陪练的家长或老师

## 3. 输入与拆解

- 页面提供一个多行文本输入框，允许输入一句中文。
- 用户可直接使用系统键盘的语音输入。
- 页面提供浏览器语音输入按钮；若浏览器支持 Web Speech API，则可以尝试直接录音转文字。
- 点击“拆成单字”后，系统仅提取中文汉字字符，忽略标点、空格和其他非汉字字符。
- 系统将句子拆成有顺序的单字列表，并保留原句用于展示。

## 4. 单字卡片区

- 每个汉字以卡片形式展示。
- 卡片上显示小写拼音和汉字本身。
- 点击拼音可朗读对应汉字。
- 点击汉字卡片可切换当前练习字，并自动朗读该字。
- 当前选中的汉字卡片应有明显高亮状态。

## 5. 当前字展示区

- 展示当前汉字的小写拼音。
- 展示当前汉字的大字形。
- 展示当前字在整句中的序号。
- 展示当前字的笔画数。
- 使用 `cnchar-draw` 在书写区域中展示笔顺。

## 6. 书写交互

- 提供“下一笔”按钮，手动逐笔展示当前字。
- 手动点击“下一笔”时，应先朗读当前这笔的笔画名称，再展示该笔。
- 提供“播放当前字”按钮。
- 点击“播放当前字”时，应先朗读当前字，再按笔顺逐笔播放。
- 自动播放当前字时，每一笔开始前应朗读该笔的笔画名称。
- 若笔画名称存在多个别名，系统随机选择一个名称朗读。
- 当前字完整播放结束后，书写区域恢复为可重新逐笔播放的初始状态。

## 7. 整句播放

- 提供“连续播放整句话”按钮。
- 点击后按句中顺序逐字播放书写过程。
- 每开始一个新字时，先朗读该字。
- 每开始一笔时，朗读当前笔画名称。
- 每个字写完后应停留约 700ms，再继续下一个字。
- 用户再次点击按钮时，应停止整句播放。

## 8. 切字交互

- 支持左右滑动切换前后字。
- 在 desktop 上提供“上一个字”“下一个字”按钮。
- 在 desktop 上支持键盘左右方向键切换。
- 手动切换当前字时，应自动朗读该字。
- 手动切换当前字时，应停止整句播放。

## 9. 朗读规则

- 优先使用 `cnchar.voice.speak` 进行朗读。
- 若 `cnchar` 语音接口不可用，则回退到浏览器原生 `speechSynthesis`。
- 当前朗读速度为 0.75 倍速。

## 10. 语音输入状态

- 语音输入按钮需要区分“未开始”和“正在监听”两种状态。
- 点击开始后，页面应给出明确状态提示，例如请求权限、开始说话、结束录音、识别成功、未识别到内容等。
- 如果浏览器不支持语音识别，应给出降级提示，建议用户使用系统键盘语音输入。
- 如果浏览器立即结束录音且没有结果，应提示浏览器语音识别服务可能不可用。

## 11. 技术约束

- 使用 Vite 作为项目启动与构建工具。
- 应用代码位于 `src/` 目录。
- 使用 `cnchar` 及其相关插件完成拼音、笔画、书写和朗读能力。
- 当前通过 CDN 引入 `cnchar`、`cnchar-poly`、`cnchar-order`、`cnchar-draw`、`cnchar-voice`。
